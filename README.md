# MVTool TUI

Инструмент на базе Rust для автоматизации рабочего процесса, управления состоянием проекта и выполнения вспомогательных сценариев через интерактивный текстовый интерфейс (TUI).

## 🎮 Навигация и управление

Программа поддерживает клавиатурный ввод.

* **Перемещение между блоками**: Используйте клавишу `Tab` для циклического переключения между секциями вперед (Проекты → Конфигурации → Компоненты → Скрипты → Консоль) или `Tab + Shift` для переключения в обратном порядке.
* **Горизонтальные списки**: Выбор активного проекта осуществляется стрелками влево **`←`** и вправо **`→`**.
* **Вертикальные списки**: Навигация по списку компонентов конфигураций или доступных скриптов выполняется стрелками вверх **`↑`** и вниз **`↓`**.
* **Прокрутка**: Если список не помещается в окно, используйте **колесо мыши** для быстрой прокрутки содержимого. (Поддерживается не всеми терминалами)
* **Выбор и активация**: **`Space`**, чтобы отметить флаг или выполнить скрипт из списка.
* **Массовое выделение**: **`Ctrl + a`**, чтобы мгновенно отметить все доступные чекбоксы при множественном выборе.
* **Выполнение**: Клавиша **`F1`** запускает основной процесс выполнения сценария на основе выбора.

## ⚙️ Описание параметров конфигурации

Программа управляется через файл `settings.json`.

### Настройки проекта и конфигураций
* **`destination_path`**: Конечная директория, куда будут копироваться артефакты **проекта**.
* **`source_path`**: Папка-источник исходных файлов (например, папка `debug` или `release`).
* **`clean_destination`**: При значении `true` программа автоматически удаляет все файлы из папки назначения по списку **компонент** перед копированием выбранных.
* **`extension_mask`**: Фильтр расширений. Программа взаимодействует только с файлами, указанными здесь (например, `*.exe`, `*.dll`), защищая остальные данные от случайного удаления/копирования.
* **`components`**: Список модулей проекта. Позволяет выбрать для обновления только конкретные части системы.
* **`scripts`**: Пользовательские команды. Выполняются в отдельном интерактивном окне.

Файл настроек должен располагаться в одной директории с исполняемым файлом. Ниже представлен эталонный пример структуры:
```json
{
  "projects": [
    {
      "name": "project1",
      "selected": true,
      "destination_path": "C:\\project1\\middleframe",
      "configures": [
        {
          "name": "Debug",
          "source_path": "C:\\project1\\debug",
          "selected": true,
          "clean_destination": true,
          "extension_mask": ["*.exe", "*.pdb", "*.dll"],
          "components": [
            { "name": "qort", "selected": true }
          ],
          "scripts": [
            { "name": "Hello Rust", "command": "echo Hello, Rust!" }
          ]
        },
        {
          "name": "Release",
          "source_path": "C:\\project1\\release",
          "selected": false,
          "clean_destination": false,
          "extension_mask": ["*.exe", "*.pdb", "*.dll"],
          "components": [
            { "name": "qort", "selected": true }
          ],
          "scripts": []
        }
      ]
    }
  ]
}
```
